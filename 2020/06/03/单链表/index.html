起始链表本来就抽象，光看代码，真的很难看出答案来，我们可以利用IDE的调试功能查看，在此之前，我们先分析一下代码

n0初始化之后，链表的结构时只有一个节点n0，n0.data=null，no.next=null，注意n0是无参构造器初始化的。

n1初始化以后，链表的就有两个节点了，n0、n1，此时n0.data=null，n0.next=n1,n1.data=1,ni.next=null,

n2初始化以后，我们先分析n2，从构造器一步一来

  1. n2 = new Node(2,n0);
  2. data = 2;
  3. next = n0.next，由n1初始化之后我们知道n0.next = n1;所以next = n1;
  4. before.next即n0.next ,this也就是n2这个节点，before.next = this的意思也就是说n0.next = n2;

所以现在n2节点是:n2.data = 2,n2.next = n1;整个链表的结构是n0.data = null,n1.next = n2,n2.data = 2,n2.next = n1,ni.data = 1,n1.next = null，也即是上面图显示那样。

仔细回顾一下代码，我们可以发现

Node<Integer> n1 = new Node(new Integer(1),n0)实现了链表添加元素的一种方式，当然这是每次只是在n0的后面添加节点，

如果要在指定位置添加节点，我们还需要做修改一下代码，先把我们的需求抽象一下我们其实是要实现这样的方法add(int index,E newItem)，就是在指定索引处添加一个元素，大致如图所示，用图片模型表示比较直观

myLinkdedList(1,itm),表示在索引1处添加一个itm，我们的思路应该是这样的

我们原来节点里是没有index的，必须在节点里添加一个index，为了以后更好的维护节点，我们遵循程序设计的单一职责的原则，重新设计一个具有索引的单链表

  • 首先应该判断这个索引是否越界或者合法，也就是判断index是否大于0或者小于size

  • 然后查找到我们要添加的索引位置的目标节点

    • 查找目标节点之前我们要先定义一个目标节点，因为后面要用到

      • 怎么定义目标节点，由上面的链表定义我们知道，所有的节点都是从head派生而来的，我们只需要创建一个新的Node，然后将head赋值给它就够了

        这里有个问题，我们所有的依据都是前面的Node的定义和链表的设计，如果我们上面的设计和定义有问题，后面不就都是错的吗？

        从逻辑上讲确实是这样的，但是事实确是，我们发现了我们定义的那种链表可以实现节点的添加，然后我们这种定义，去推广出我们在指定索引位置添加元素，结果是切实可行的。我们确实可以用我们自己设计的链表进行元素的添加，这在实践上是正确的。

    • 这个查找目标节点并没有很好地办法，我们只能一个一个遍历

  • 将当前要添加的节点插到目标节点前面

    将当前节点插到目标节点的前面，其实上面节点里已经实现了在指定节点处添加新节点，当目标节点后面没有节点就在目标节点后面添加节点，当目标节点后面有元素，就在目标节点前面添加元素，因此如果要在目标节点前面添加元素，我们要么在目标节点前节点的后面添加，要么将目标节点移动到当前位置的后面，然后再在当前位置添加。当然后面一种方式比较简单，因为我们更容易找到目标节点的位置，然后把它移动到后面即可

    然后将新节点插到目标节点的前面

最终的add()方法就是这样的；
