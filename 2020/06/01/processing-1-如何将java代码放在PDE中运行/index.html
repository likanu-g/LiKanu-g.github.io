processing---1---如何将java代码放在PDE中运行

 |
2020-06-01 (Updated: 2020-06-04)
java
processing

前面的基础部分，我们已经学习了processing库用java开发工具开发，对于processing的基本原理，模板方法想必已经有了足够的认识

我也说过为什么选择用基本的java开发工具而不用PDE的原因，但是对于那些不熟悉java代码，不懂java开发环境搭建的同学来说，他们觉得PDE操作简单，上手快，不需要额外的配置，不用下载什么gradle这种比较专业的构建工具等。

那么既然processing是java开发的，我们怎么把java代码直接放在PDE中运行呢，由前面的学习，我们发现，PDE中基本没有我们出现我们在VSCode中写的class这种字样，只有setup(),draw()等方法，也没有我们在练习中写的main方法。并且在PDE中写的东西都是放在一个文件里，而我们在VSCode中写了两个类，一个App类，一个Block类。

下面就讲怎么将我们写的java APP放在PDE中运行。

下图是PDE的界面布局，我们在Text Editor中编写processing代码

具体步骤如下：

  1. 将我们写的App.java代码全部复制到Text Editor中

  2. 删除第一行的包名，删除public class App extends PApplet{类申明以及最后一行的大括号

  3. 将构造器中初始化的变量复制到setup()中初始化

  4. 删除构造器

  5. 复制Block.java的内容粘贴到PDE的最下面，作为原项目的内部类，不需要做其他修改，删除Block的包名即可

  6. 保存PDE中的代码，自定义一个文件夹，打开这个文件夹在里面新建一个data的文件夹，就是在那个.pde文件所在的同级目录下

  7. 将VSCode项目中resource文件夹里的资源文件复制到data文件夹下面

  8. 点击PDE的运行，可以正常运行程序，没有报错，说明代码修改正确

在此不建议将PDE中的代码复制到java项目中运行，一般是不能正常运行的。
