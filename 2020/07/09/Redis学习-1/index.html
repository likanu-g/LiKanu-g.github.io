string(字符串)

string的基本概念

Redis的字符串是动态字符串，是可以修改的字符串，类似java的ArrayList。

string的基本操作

新增/修改

  • 单个新增/修改：set命令

  • 批量新增/修改：mset

自增计数

自增命令 incr、incrby

查询

  • 单个查询：get命令

  • 批量查询：mget命令

  • 是否存在：exists命令

删除

删除命令：del

过期

  • 一般过期：expire

  • set拓展过期：setex

list(列表)

基本概念

Redis的列表相当于Java里的linkedList，底层是链表，当列表弹出了最后一个元素之后，该数据结构自动被删除，内存被回收。

Redis的列表结构常用来做异步队列使用。将需要延后处理的任务结构体序列化成字符串塞进Redis的列表，另一个线程从这个列表种轮询数据进行处理。

基本操作

队列：右边进左边出

  • 入队：rpush

  • 出队：lpop

  • 队列长度：llen

栈：右边进右边出

  • 入栈：rpush

  • 出栈：rpop

慢操作

  • lindex：根据索引查找元素

  • lrange ：遍历列表

  • ltrim：截取列表

hash(哈希)

基本概念

Redis的字典相当于hashMap，无序，Redis为了高性能，不能堵塞服务，所以采用渐进式rehash策略。

渐进式rehash会在rehash的同时，保留新旧两个hash结构，查询时会同时查询两个hash结构，然后在后续的定时任务中以及hash的子指令中，循序渐进地将旧hash的内容一点点迁移到新的hash结构中。

基本操作

增加元素：
单个元素增加hset
批量增加元素hmset
查询元素
通过key获取value：hget
遍历hash：hgetall
返回hash容量：hlen
计数：hincrby

set(集合)

基本概念

Redis的集合相当于Java中的hashSet，它内部的键值对是无序的唯一的。

当集合中最后一个元素被移除后，数据结构自动删除，内存被回收。

基本操作

增加元素：sadd
删除元素：spop
查询元素
遍历set：smembers
判断元素是否存在：sismember
统计集合大小：scard

zset(有序集合)

基本概念

Redis的zset类似java的SortedSet和hashMap的结合体，无重复的有序hash。它的内部实现用的是一种叫做“跳跃列表”的数据结构。

基本操作

添加元素：zadd

遍历集合
正序遍历：zrange

逆序遍历：zrevrange

根据分值区域遍历：zrangebyscore

根据分值区域遍历返回分值：zrangebyscore

获取指定value的score：zcore

计算集合容量：zcard
排序
排名：zrank
删除元素：zrem

容器型数据结构

容器数据结构

list/set/hash/zset 这四种数据结构是容器型数据结构，它们共享下面两条通用规则

  1. 容器不存在，就创建一个，再进行操作。
  2. 如果容器里元素没有了，那么立即删除元素，释放内存。

过期时间

Redis所有的数据结构都可以设置过期时间，时间到了，Redis会自动删除相应的对象。
